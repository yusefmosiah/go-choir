Key Comments Confirming VAL-VM-011 Compliance:
----------------------------------------------

1. Header comment:
"The guest image includes ONLY the sandbox runtime binary and its
minimum runtime dependencies. Provider credentials, gateway secrets,
auth signing keys, and other host-side material are explicitly
excluded from the guest (VAL-VM-011)."

2. guestRoot derivation:
"Minimal guest root filesystem containing only the sandbox runtime.
This is the set of files that will be packed into the ext4 rootfs image.

Includes: sandbox binary, init script, and minimal networking tools
(ip from iproute2, plus standard POSIX utilities from coreutils/shadow).
Provider credentials, gateway secrets, auth signing keys, and other
host-side material are explicitly excluded (VAL-VM-011)."

3. runCommand comment:
"Explicitly do NOT pass any secret-containing environment variables
into the derivation. The Nix sandbox prevents access to host
environment anyway, but this comment makes the intent clear (VAL-VM-011)."

4. buildFirecrackerConfig (internal/vmmanager/manager.go):
"Build the Firecracker configuration.
Provider credentials are explicitly NOT included here (VAL-VM-011)."

"Build the guest kernel boot arguments.
These contain NO provider credentials (VAL-VM-011)."

Guest Init Script Analysis:
--------------------------
The guest init script (extracted to guestInitScript derivation):

1. Reads kernel cmdline for: guest_port, vm_id, epoch, ip
2. Sets RUNTIME_GATEWAY_URL from ip= parameter (e.g., http://172.X.0.1:8084)
3. Reads gateway token from /mnt/persistent/gateway-token
4. Sets SANDBOX_PORT, SANDBOX_ID from cmdline
5. Executes /bin/sandbox

INIT SCRIPT DOES NOT:
- Read ZAI_API_KEY or FIREWORKS_API_KEY
- Access any provider credentials
- Load any host-side secrets

Conclusion: The guest image is explicitly designed to exclude all provider
credentials. The build system (Nix) enforces this at build time by:
1. Not including credential files in the derivation
2. Using writeShellScript to isolate build scripts
3. Explicit comments marking VAL-VM-011 compliance points
